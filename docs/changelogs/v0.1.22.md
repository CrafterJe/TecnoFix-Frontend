### Corregido
- **La notificación de actualización ya no se pierde si el `check()` termina antes del splash.** Había un race condition: si el chequeo de red completaba antes de que el `UpdaterDialog` se montara (después del splash de ~2s), el estado se emitía a un callback `null` y el dialog nunca aparecía. Ahora el estado pendiente se almacena y se emite en cuanto el callback se registra.
